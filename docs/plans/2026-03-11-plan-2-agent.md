# Plan 2: Relix Agent (`relixctl`)

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the CLI agent that runs on developer machines, discovers Claude Code sessions, bridges them to the relay, and handles E2E encryption.

**Architecture:** Go CLI using cobra for commands, nhooyr.io/websocket for relay connection, os/exec to spawn `claude -p --output-format stream-json` for headless sessions. Runs as a background daemon via launchd (macOS) / systemd (Linux).

**Tech Stack:** Go 1.23+, cobra, nhooyr.io/websocket, github.com/relixdev/protocol

**Spec:** `docs/specs/2026-03-11-relix-design.md`
**Depends on:** Plan 1 (protocol module)

## Task Outline

### Chunk 1: CLI Skeleton & Config (Tasks 1-4)

- [ ] **Task 1:** Init Go module, add dependencies (cobra, websocket, protocol)
- [ ] **Task 2:** Config module — read/write `~/.relixctl/config.json` (relay URL, machine ID, auth token)
- [ ] **Task 3:** Cobra root command + `config set` subcommand
- [ ] **Task 4:** Key store — generate, save, load X25519 key pairs to `~/.relixctl/keys/`

### Chunk 2: Claude Code Adapter (Tasks 5-8)

- [ ] **Task 5:** Session discovery — scan running processes for `claude` instances, read `~/.claude/projects/` for session files
- [ ] **Task 6:** Claude Code adapter — spawn `claude -p --output-format stream-json --verbose`, parse NDJSON events into protocol.Event stream
- [ ] **Task 7:** Claude Code adapter — send user input and approval responses via stdin
- [ ] **Task 8:** Integration test — adapter discovers a mock session, streams events, accepts input

### Chunk 3: Relay Connection (Tasks 9-12)

- [ ] **Task 9:** WebSocket client — connect to relay, authenticate with JWT, send/receive Envelopes
- [ ] **Task 10:** Reconnection logic — exponential backoff (1s→60s), auto-reconnect on network changes
- [ ] **Task 11:** Message routing — encrypt outbound events with SealPayload, decrypt inbound with OpenPayload
- [ ] **Task 12:** Machine status updates — send online/offline/active status to relay

### Chunk 4: Auth & Pairing (Tasks 13-16)

- [ ] **Task 13:** OAuth login flow — open browser, run local callback server, exchange code for JWT
- [ ] **Task 14:** Device code login — poll-based flow for headless servers (no browser)
- [ ] **Task 15:** Pairing flow — receive 6-digit code, exchange public keys via relay, SAS verification
- [ ] **Task 16:** Token refresh — auto-refresh JWT before expiry

### Chunk 5: Daemon & Service (Tasks 17-20)

- [ ] **Task 17:** Daemon main loop — discover sessions, connect to relay, bridge events
- [ ] **Task 18:** `relixctl start` / `relixctl stop` — manage daemon process
- [ ] **Task 19:** launchd plist generation + install (macOS)
- [ ] **Task 20:** systemd unit generation + install (Linux)

### Chunk 6: Remaining Commands & Polish (Tasks 21-24)

- [ ] **Task 21:** `relixctl status` — show connection state, active sessions, relay latency
- [ ] **Task 22:** `relixctl sessions` — list Claude Code sessions with status
- [ ] **Task 23:** `relixctl uninstall` — stop daemon, remove service, delete keys/config
- [ ] **Task 24:** Cross-compile builds with Goreleaser, test on macOS + Linux

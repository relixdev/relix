# Plan 3: Relix Relay

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the open-source WebSocket relay server that routes E2E-encrypted messages between agents and mobile clients.

**Architecture:** Stateless Go HTTP server (net/http + nhooyr.io/websocket). In-memory connection registry maps user/machine IDs to WebSocket connections. JWT validation on connect. Message buffering for disconnected clients. Docker image for deployment.

**Tech Stack:** Go 1.23+, nhooyr.io/websocket, golang-jwt/jwt/v5, github.com/relixdev/protocol

**Spec:** `docs/specs/2026-03-11-relix-design.md`
**Depends on:** Plan 1 (protocol module)
**License:** MIT (open source)

## Task Outline

### Chunk 1: Server Skeleton & Connection Handling (Tasks 1-5)

- [ ] **Task 1:** Init Go module, add dependencies
- [ ] **Task 2:** Config module — port, JWT secret (from env vars), buffer limits
- [ ] **Task 3:** WebSocket connection wrapper — read/write Envelopes, ping/pong keepalive (30s)
- [ ] **Task 4:** HTTP server — health check endpoint (`/health`), WebSocket upgrade at `/ws`
- [ ] **Task 5:** JWT auth — validate token on WebSocket connect, extract user ID and client role (agent vs mobile)

### Chunk 2: Hub & Routing (Tasks 6-9)

- [ ] **Task 6:** Connection hub — register/unregister connections, map (userID, machineID) → agent conn, userID → mobile conns
- [ ] **Task 7:** Message routing — agent→mobile: find mobile connections for user, forward envelope. mobile→agent: find agent connection for (user, machine), forward envelope.
- [ ] **Task 8:** Machine status tracking — when agent connects/disconnects, notify mobile clients
- [ ] **Task 9:** Ping/pong + dead connection cleanup — detect stale connections, unregister, notify

### Chunk 3: Buffering & Reliability (Tasks 10-12)

- [ ] **Task 10:** Message buffer — when mobile is disconnected, buffer up to 1000 messages / 10MB per machine, 24h TTL
- [ ] **Task 11:** Buffer replay — on mobile reconnect, replay buffered messages in seq order
- [ ] **Task 12:** Pairing relay — route pairing messages (public key exchange) between agent and mobile during setup

### Chunk 4: Pairing Rate Limiting (Tasks 13-14)

- [ ] **Task 13:** Pairing code validation — 5-minute expiry, max 5 attempts, 1 attempt per 3s per IP
- [ ] **Task 14:** Rate limiting middleware — per-IP connection limits, abuse prevention

### Chunk 5: Deployment & Observability (Tasks 15-18)

- [ ] **Task 15:** Prometheus metrics — active connections (by type), messages routed/sec, buffer size, latency histograms
- [ ] **Task 16:** Dockerfile — multi-stage build, minimal image, non-root user
- [ ] **Task 17:** docker-compose.yml — relay + Prometheus + Grafana for local dev
- [ ] **Task 18:** Integration tests — spin up relay, connect mock agent + mock mobile, verify message routing, buffering, reconnect replay
